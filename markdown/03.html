3. EDITS of config.yaml

3. EDITS of config.yaml

In this chapter, the work steps are with an editor application (such as Notepad). Do not touch the Python Command Prompt which we justed work with

3.1 open config.yaml

The newly created project folder task-researcher-yyyy-mm-dd should have config.yaml inside. config.yaml will be edited. Editor applications can be any, including Vscode or Notepad.

  • task-researcher-yyyy-mm-dd
    • config.yaml
    • dlc-models
    • labeled-data
    • training-datasets
    • videos

3.2 edit config.yaml

You need to edit bodyparts and numframes2pick

# --- omitted ---
bodyparts: # lines to specify
- bodypart1
- bodypart2
- bodypart3
- objectA
start: 0
stop: 1
numframes2pick: 50 # line to specify

# --- omitted ----

  • what is 'bodyparts'?

    bodyparts defines everything that needs to be tracked. They can include not only body parts, but also bars, a spout, etc. There is no limit how many bodyparts you want to create. to define an item to track,

    • name to track

      For example, if we plan to track a lowerjaw and a spout, then the code should be like the following.

      ```yaml bodyparts:

    • lowerjaw

    • spout ```

      It is recommended that bodyparts have some spatial order in the video. For instance the 1st bodyparts is from the top, and the last from the bottom.

  • what is numframes2pick?

    numframes2pick specifies a number of frames to label as a training dataset. Defualt number is 20. The larger the numframes2pick is, the better model the network can develop. In particular, larger numframes2picktracking is necessary for a body part which rarely shows up (e.g., tongue). On the other hand, the larger numframes2picktracking means more effort for labeling.

    Here we will proceed with the default number, 20.

      numframes2pick: 20

3.3 overwrite 'config.yaml'

After all the edits are done, save (overwrite) config.yaml. You can close the editor application here.
