2. CREATION OF A PROJECT

All of the steps in this chapter is in the Python Command Prompt

2.1 Start Python

Run the fllowing code

python

2.2 load deeplabcut libraries

Run the fllowing code. It may take time.

import deeplabcut

2.3 Create a project

Run the following code, after modifying 'task_name', 'researcher_name', 'video_path_list'

This code will creates, within your working direcotry, a project folder named task-researcher-yyyy-mm-dd.

Warning : function deeplabcut.create_new_project() can get input copy_videos as False or True. This gitbook chooses False, which does not copy the video, but instead create a link file. The link file requires an additional step later to upload a video to Google Colab.

Instead of False, you could choose True for input copy_videos. This choice will copy a video in the working directory. 

# lines to edit ---
# name of the task
task_name = 'task'

# name of one who does labeling
researcher_name = 'researcher'

# paths to vidoe files
video_path_list = [ 'C:\\*****\\video1.avi', 'C:\\*****\\video2.avi' ] # when you have two videos
#video_path_list = [ 'video1.avi' ] # when one video. The format still needs to be [];

# lines not to edit ---
# creation of a project
deeplabcut.create_new_project( task_name , researcher_name, video_path_list, copy_videos = True )

# save paths (for later)
import os; import datetime;
todays_date = datetime.datetime.now().strftime('%Y-%m-%d')
path_project = os.getcwd() + f'\\{task_name}-{researcher_name}-{todays_date}'
path_config = path_project + "\\config.yaml"

The directory structure of this specific example should be the following.

  • C drive
    • Users
      • UserName
        • Documents
          • DeepLabCut
            • task-researcher-yyyy-mm-dd
              • config.yaml
              • dlc-models
              • labeled-data
              • training-datasets
              • videos
